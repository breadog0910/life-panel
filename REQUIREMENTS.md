# 📋 需求清单 & 约束文档

> 每步开发前对照此文档，确保不偏离用户需求

## 核心闭环（用户原文）
- 桌面悬浮窗：一句话复盘「今天学到了什么」+ 时间安排 + 计时
- 网页端：同步展示时间安排 + 日记 + 今日心情/感受/感悟
- 悬浮伙伴点击 → 复盘输入、看计时、提醒输出
- 手机 / 电脑网页同步

## 约束条件
1. ✅ **Web 应用**，浏览器访问，手机电脑都能用
2. ✅ **数据云端同步**（Supabase），多设备实时互通
3. ✅ **桌面悬浮窗**（Electron），透明置顶，角色 + 功能面板
4. ✅ **Git 备份**：每做完一步就 commit，写清楚备注
5. ✅ **开发者日志**：每步记录到 DEVLOG.md

## 布局规范
- **桌面端**（≥768px）：左侧固定导航（200px）+ 右侧内容区
- **移动端**（<768px）：卡片堆叠 + 底部 5 tab 导航
- **默认首页**：今日概览

## 悬浮伙伴规范
- **待机**：透明置顶窗口，角色 GIF，偶尔冒泡
- **提醒**：表情变化 + 气泡文字 + Windows 通知
- **交互**：点击展开面板（番茄钟 + 一句话复盘 + 心情 + 快捷按钮）
- **右键菜单**：打开网页 / 设置 / 更换角色 / 退出
- **拖拽移动**：位置持久化到 partner_config 表
- **数据通道**：复盘内容 → Supabase reflections 表（Realtime）

## 技术栈（已确认）
- Next.js 14 App Router + React + TypeScript
- Tailwind CSS + shadcn/ui（base-nova 风格）
- Supabase（认证 + PostgreSQL + Realtime + RLS）
- Python tkinter（桌面悬浮窗，替代 Electron）
- Vercel 部署（Web 端）

## 开发路线
1. **第 1 步** ✅：Web 面板 → 初始化项目 → Supabase → 认证 → 布局 → 核心页面 → 部署
2. **第 2 步** ✅：Python tkinter 悬浮伙伴（透明窗口 + 三模式角色 + 设置面板）
3. **第 3 步** ✅：串联闭环 + 补充 P1/P2 模块（记账/目标/提醒/统计/伙伴自定义）
